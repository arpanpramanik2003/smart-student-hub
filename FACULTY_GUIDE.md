## âœ… Review Queue

The heart of the faculty workflow - reviewing pending student activities.

### Features

#### 1. **Activity Cards**
Each submission shows:
- Student name and ID
- Activity type (workshop, conference, certification, etc.)
- Activity title and description
- Requested credits (student's request)
- Date of activity
- Submission timestamp
- Attached certificate/proof

#### 2. **Review Actions**

**To Approve:**
1. Click **Review** button on an activity
2. Review the activity details carefully
3. Check the attached certificate
4. Adjust credits if needed (0-10 range)
5. Add optional remarks/feedback
6. Click **Approve**

**To Reject:**
1. Click **Review** button
2. Add remarks explaining why (required for rejection)
3. Click **Reject**

#### 3. **Filtering**
- **By Status**: Pending, Approved, Rejected
- **By Department**: Filter by student's department
- **Pagination**: Navigate through multiple pages

### Best Practices for Review

âœ… **Be Fair**: Assign credits based on activity significance  
âœ… **Provide Feedback**: Add constructive remarks

## ğŸ”§ Troubleshooting

### Common Issues

#### Can't See Pending Activities
**Solution:**
- Check if you're on the correct tab
- Refresh the page
- Verify your faculty account status

#### Certificate Won't Load
**Solution:**
- Check internet connection
- Try different browser
- Report if issue persists

#### Profile Picture Not Showing in Student List
**Solution:**
- Images are loaded from Cloudinary CDN
- Check if student uploaded a picture
- Refresh the page
- Clear browser cache

#### Search Not Working
**Solution:**
- Ensure search term is correct
- Try different search criteria
- Check filters are not too restrictive

#### Pagination Issues
**Solution:**
- Use Previous/Next buttons
- Don't refresh during navigation
- Report if page numbers are incorrect
